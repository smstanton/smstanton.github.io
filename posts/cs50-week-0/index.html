CS50 Week 0

  • April 1, 2019
  • CS50  

While on the road for work last week, I watched the Week 0 lecture and completed the problem set. This was probably the 3rd or 4th time I’ve watched this lecture, since I’ve started and not completed CS50 several times. CS50 re-records the lecture each year for the live class at Harvard, and online students like myself get the benefit of updated materials. The content is pretty consistent from year to year, but the execution continues to improve (impressive, considering how good it is to begin with). The production quality of this class is top-notch, as is David Malan’s delivery. In my assessment, if this first lecture is any indication about the quality of this course for beginners, CS50 truly is the right starting point for driven individuals to begin their CS journey.

The problem set for this week is to build a program using MIT’s Scratch. Scratch is a lot of fun to work with as you are moving puzzle pieces around to build programs - and Scratch helps you understand what your options are by only allowing puzzle pieces that logically fit together to actually fit together on the screen. Once you get the general feel for what to do and where to find things, programming with Scratch feels less like programming and more like playing with a children’s science kit. All exploration, minimal frustration.

The assignment is pretty open - designating a minimum number of Scratch elements such as sprites, scripts, conditions, sounds, etc. I built one program with Scratch a few years ago, but decided to do a new one this year as a refresher. Here is a link to the submission from a few years ago which you can play until you reach your own limitations. You can click the “See Inside” link on either submission to see how it was built.
